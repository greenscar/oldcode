Last :- A "LAST PAGE" tag

Name : last.pl

Version number: 1.00 (22Aug95)

Release Date : 22 Aug 95

Servers : NCSA-style

This is a really silly little tag, but some people seem to have been asking for it!!!!

What's it do?

This script creates hyper-link tags which refer to the last Web resource that was called by the browser before the current page.

Basically, it 'fakes' the BACK button.

Why anyone wants this, when nearly all browsers already have a BACK button, I don't know.. still, that's users for you!!! *8)

How to use last.pl

1> Take a copy of the Perl script, and put it into your /cgi-bin directory.

2> Make last.pl exectuable:

chmod +x last.pl

3> Make a WebPage which contains your BACK hyper-link. This normally looks like this:

<a href="some.page.html">BACK</a>

However, you need last.pl to 'fake' the last WebResource's URL, so your tag now looks like this:

<a href="<!--#exec cgi="/cgi-bin/last.pl" -->">BACK</a>

How does last.pl work?

last.pl is called by your page by a server side include. This basically gets last.pl to retrieve and return the URL of the last Web resource your browser visited.

The returned URL is put in the place of your normal hyper-link's URL.

You can get the last.pl by clicking on any of the hi-lited links in this page, or by clicking HERE.
NOTE:- For last.pl to work, your WebServer MUST have server side include capability. This generally means that you must be using an NCSA-style server, and that server side include capability has already been correctly setup to work with your server.
For a more full reference and explanation of the back button function, look at Simulating the back button.
